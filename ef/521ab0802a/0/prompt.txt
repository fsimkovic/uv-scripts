# Add `features` (optional dependencies) support to uvs

## Context

uvs currently has no way to install optional dependency groups (extras) when running scripts. Users who define `[project.optional-dependencies]` in their `pyproject.toml` can't opt into those extras via uvs. Adding a `features` config key (mirroring Hatch's convention) lets users declare which extras to install, translating to `--extra <name>` flags on every `uv run` invocation.

## Changes

### 1. `src/uv_script/config.py` — parse `features` from config

- Add `features: list[str]` field to `UvsConfig` (default: empty list)
- In `load_config()`, after the editable parsing block (line 70), parse `features` from `uvs_table`:
  - Validate it's a list of strings (same pattern as `editable`, but **no path resolution** — these are extra names, not paths)
  - Raise `ConfigError` on invalid types
- Pass `features` to the `UvsConfig` return

Example config:
```toml
[tool.uvs]
features = ["speedups", "cli"]
```

### 2. `src/uv_script/runner.py` — add `--extra` flags to `uv run`

- Add `features: list[str] | None = None` parameter to `run_script()` and `_exec_one()`
- In `_exec_one()`, build `features_flags` list: `["--extra", name]` for each feature
- Insert flags after editable flags, before command parts:
  ```
  ["uv", "run"] + editable_flags + features_flags + parts
  ```
- Thread `features` from `run_script()` into `_exec_one()` calls

### 3. `src/uv_script/cli.py` — add `--no-features` flag

- Add `--no-features` argument (mirrors `--no-editable`)
- Compute: `features = None if parsed.no_features else (config.features or None)`
- Pass `features=features` to `run_script()`
